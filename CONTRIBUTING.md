# Contributing to Waveon

Thanks for contributing to Waveon.

## Workflow

1. Fork the repository.
2. Create a feature branch:

```bash
git checkout -b feature/short-description
```

3. Make focused changes.
4. Run checks before opening PR:

```bash
npm run typecheck
npm run build
```

5. Commit with clear messages.
6. Open a Pull Request with:
   - problem description
   - change summary
   - testing notes

## Contribution Guidelines

- Keep PRs small and scoped.
- Follow existing project style and structure.
- Update docs in [`docs/`](docs) when behavior changes.
- Avoid unrelated refactors in the same PR.

## Areas You Can Help With

- bug fixes
- playback improvements
- UX/accessibility polish
- performance optimization
- documentation improvements

## Reporting Issues

When opening an issue, include:

- environment (OS, Node version)
- reproduction steps
- expected vs actual behavior
- logs/screenshots when possible

## Conduct

By participating, you agree to follow [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
